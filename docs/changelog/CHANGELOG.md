# 更新日志

## [1.1.1] - 2025-09-10

### 修复

- 彻底解决“页面越滚越长”的问题：动画组件改为 fixed 且使用 window 尺寸，移除内容区 min-height 依赖，避免高度抖动。
- 修正动画组件客户端渲染要求，统一添加 "use client" 并在布局中以动态导入禁用 SSR。

### 优化

- 顶部左侧品牌（Logo + 标准字）等比例显示并精确垂直对齐；标题与内容区留白、居中与对齐更统一；内容区默认透明可透出背景动画。
- 全局布局 padding 与容器宽度（max-w-6xl + 响应式内边距）统一，避免文字贴边。

### 文档

- 更新动画实现与样式规范说明，记录 fixed 渲染、window 尺寸、rAF 清理与布局策略。

## [1.1.0] - 2025-09-09

### 新增

- 动画背景统一为 fixed + window 尺寸，增加 rAF 取消机制与事件清理。
- `BrandHeroText` 新增对齐选项，首页标题默认居中。
- 语言切换器固定右上角显示。

### 修改

- 根级样式移除 100vw，统一 width:100% + overflow-x:hidden。
- `.content-area` 取消 `min-height:100vh`，避免高度抖动；`layout` 使用固定上下 240px 留白。
- 所有 section 背景改为透明，露出下层动画。

### 文档

- 更新动画设计规范，记录 fixed 渲染、尺寸策略与 rAF 清理要求。

## [1.0.1] - 2024-06-13

### 修复

- 自动修复动画 canvas 作为内容区背景，canvas 高度与内容区同步，线条分布均匀，动画始终作为所有内容的背景涂层。
- 移除 Hero 区块图片，避免图片加载失败。
- 添加内容区占位，确保容器有高度，canvas 能正常显示。
- 修复 Section 数据健壮性，避免内容区渲染异常导致页面空白。
- 检查并优化 useEffect 动画逻辑，避免死循环和页面无限拉伸。
- 提醒：后续每次自动修正都需记录到 CHANGELOG.md，避免重复修正和低级错误。

## [1.0.2] - 2024-06-13

### 架构升级

- 动画背景由 canvas 动态绘制切换为 SVG 静态背景，彻底避免死循环、无限拉伸、SSR/CSR 问题。
- 移除所有图片引用，logo 只用纯文本。
- 内容区结构极简清晰，SVG 绝对定位，内容区用 relative z-10 包裹。
- 线条分布均匀，视觉极简现代。
- 本次为重大架构升级，后续动画建议优先用 SVG/CSS 层实现。

## [1.0.3] - 2024-06-13

### 修复

- canvas 动画背景结构调整为只包裹实际内容区，彻底解决页面被越推越长的问题。
